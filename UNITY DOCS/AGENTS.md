# Unity Project Documentation

This folder contains recommended AI assistant rules and documentation sourced from the **agent-rules** repository (MIT License). Copy any `.mdc` file into your Unity project's `.cursor/rules/` directory to enable the associated workflow or guideline.

## Included Files
- `commit.mdc` – Conventional commit workflow with pre-commit checks.
- `add-to-changelog.mdc` – Structured updates to `CHANGELOG.md`.
- `check.mdc` – Comprehensive code quality and security checks.
- `clean.mdc` – Fix formatting and linting issues.
- `code-analysis.mdc` – Options for deep code analysis.
- `bug-fix.mdc` – End-to-end bug fix process.
- `implement-task.mdc` – Methodical task implementation steps.
- `pr-review.mdc` – Multi-perspective pull request review checklist.
- `create-docs.mdc` – Steps for generating thorough documentation.
- `mermaid.mdc` – Generate Mermaid diagrams for visualizing systems.
 - `screenshot-automation.mdc` – Unity C# patterns for automated screenshots.

## Usage
1. Place desired `.mdc` files in your project's `.cursor/rules/` directory.
2. The frontmatter in each file controls when the rule is applied.
3. Use these guidelines to standardize workflows across your Unity project.
